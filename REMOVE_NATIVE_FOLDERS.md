# Инструкция по удалению нативных папок

## Зачем удалять?

Expo автоматически генерирует нативные папки `android/` и `ios/` при сборке. Наличие этих папок в проекте создает конфликт с конфигурацией Expo.

## Что делать:

### 1. Удалите папки (вручную или через команду):

**Через PowerShell:**
```powershell
Remove-Item -Recurse -Force android
Remove-Item -Recurse -Force ios
```

**Или вручную:**
- Удалите папку `android/`
- Удалите папку `ios/` (если есть)

### 2. Проверьте результат:

```bash
npx expo-doctor
```

Должно быть **0 ошибок**!

### 3. Соберите APK:

После удаления папок соберите APK через EAS Build:

```bash
eas build --platform android --profile preview
```

Expo автоматически создаст правильные нативные папки с нужными настройками.

## Важно:

- ✅ Нативные папки будут автоматически созданы при сборке
- ✅ Все настройки из `app.json` будут применены
- ✅ Не нужно вручную настраивать Gradle или Xcode
- ✅ Проект станет чище и проще в управлении

## Если нужны нативные папки локально:

Если вам нужны нативные папки для локальной разработки:

```bash
npx expo prebuild
```

Эта команда создаст `android/` и `ios/` папки на основе конфигурации из `app.json`.
