# Удаление нативных папок для Expo

## Проблема

Expo-doctor предупреждает, что проект содержит нативные папки (`android/`, `ios/`), но также имеет конфигурацию Expo в `app.json`. Это создает конфликт.

## Решение

Для работы с Expo рекомендуется удалить нативные папки. Expo будет генерировать их автоматически при необходимости через:
- `npx expo prebuild` - для локальной разработки
- EAS Build - для облачной сборки

## Вариант 1: Удалить нативные папки (рекомендуется)

### Шаг 1: Удалите папки

```bash
# Удалите папку android
Remove-Item -Recurse -Force android

# Удалите папку ios (если есть)
Remove-Item -Recurse -Force ios
```

### Шаг 2: Обновите .gitignore

Добавьте в `.gitignore`:

```
# Нативные папки (генерируются автоматически Expo)
android/
ios/
```

### Шаг 3: Проверьте

```bash
npx expo-doctor
```

Теперь должно быть 0 ошибок!

## Вариант 2: Оставить нативные папки

Если вы хотите оставить нативные папки (например, для кастомных нативных модулей):

1. Удалите нативные настройки из `app.json`:
   - `orientation`
   - `userInterfaceStyle`
   - `splash`
   - `ios`
   - `android`
   - `plugins`

2. Настройте все в нативных файлах напрямую:
   - `android/app/build.gradle`
   - `android/app/src/main/AndroidManifest.xml`
   - `ios/...`

**Не рекомендуется** для Expo проектов, так как теряются преимущества Expo.

## Рекомендация

**Используйте Вариант 1** - удалите нативные папки. Expo автоматически создаст их при сборке через EAS Build или при запуске `npx expo prebuild`.

## После удаления

1. Соберите APK через EAS Build:
   ```bash
   eas build --platform android --profile preview
   ```

2. Или создайте нативные папки локально (если нужно):
   ```bash
   npx expo prebuild
   ```

Expo автоматически создаст правильные нативные папки с нужными настройками!
