# Исправление предупреждения expo-doctor

## Проблема

`expo-doctor` показывает предупреждение:
```
✖ Check for common project setup issues
The .expo directory is not ignored by Git.
```

## Решение

`.expo/` уже добавлен в `.gitignore`, но `expo-doctor` требует инициализированный Git репозиторий для проверки.

### Шаг 1: Инициализируйте Git репозиторий

Выполните в PowerShell:

```powershell
git init
```

### Шаг 2: Проверьте результат

```bash
npx expo-doctor
```

Теперь должно быть **0 ошибок**!

## Альтернативное решение

Если вы не хотите использовать Git, можете игнорировать это предупреждение. Оно не критично и не влияет на работу приложения.

`.expo/` уже правильно добавлен в `.gitignore`, и когда вы инициализируете Git репозиторий, эта папка будет игнорироваться.

## Проверка

После инициализации Git репозитория:

```bash
npx expo-doctor
```

Ожидаемый результат:
```
✅ All checks passed!
```
