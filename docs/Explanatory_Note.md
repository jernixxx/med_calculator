## 1. ВВЕДЕНИЕ

Данная пояснительная записка является документом, описывающим разработанное мобильное приложение "BMR/TDEE Калькулятор" - специализированный медицинский инструмент для расчета энергетических потребностей организма.

Работа выполнена в рамках курсового проектирования по дисциплине "Мобильные приложения" и включает в себя полный цикл разработки от проектирования до реализации.

### 1.1 Назначение приложения

Приложение предназначено для:
- Врачей-диетологов и нутрициологов
- Пациентов, следящих за питанием и весом
- Фитнес-тренеров
- Людей, ведущих здоровый образ жизни

### 1.2 Основные функции

- Расчет базального метаболизма (BMR) по медицинским формулам
- Расчет суточной потребности в энергии (TDEE)
- Предоставление персонализированных рекомендаций
- Сохранение истории расчетов
- Отслеживание динамики изменений

## 2. АКТУАЛЬНОСТЬ ПРОЕКТА

### 2.1 Медицинский аспект

В современном мире проблемы избыточного веса и ожирения становятся все более актуальными. По данным ВОЗ, более 1.9 миллиарда взрослых имеют избыточный вес, из них более 650 миллионов страдают ожирением.

Правильный расчет энергетических потребностей организма является основой для:
- Эффективного снижения веса
- Набора мышечной массы
- Поддержания здорового веса
- Профилактики заболеваний

### 2.2 Технологический аспект

Мобильные приложения обеспечивают:
- Доступность - расчеты всегда под рукой
- Удобство - интуитивный интерфейс
- Персонализация - учет индивидуальных параметров
- История - отслеживание прогресса

### 2.3 Преимущества разработанного решения

- **Научная основа** - используются медицински признанные формулы
- **Оффлайн работа** - не требуется постоянное подключение к интернету
- **Локальное хранение** - данные не передаются третьим лицам
- **Подробная интерпретация** - не просто цифры, а рекомендации
- **Русский язык** - полностью локализованный интерфейс

## 5. АРХИТЕКТУРА ПРИЛОЖЕНИЯ

### 5.1 Общая архитектура

Приложение построено по принципу **трехслойной архитектуры**:

```
┌─────────────────────────────────────┐
│     Presentation Layer              │
│  (UI Components & Screens)          │
├─────────────────────────────────────┤
│     Business Logic Layer            │
│  (Services & Calculations)          │
├─────────────────────────────────────┤
│     Data Access Layer               │
│  (Database & Storage)               │
└─────────────────────────────────────┘
```

### 5.2 Слой представления (Presentation Layer)

**Компоненты:**
- `HomeScreen` - главный экран приложения
- `CalculationScreen` - форма ввода параметров
- `ResultScreen` - отображение результатов
- `HistoryScreen` - история расчетов

**Переиспользуемые UI компоненты:**
- `Button` - кнопки различных стилей
- `InputField` - текстовые поля ввода
- `Card` - карточки для группировки контента
- `ResultCard` - карточки для отображения результатов
- `HistoryItem` - элемент списка истории

### 5.3 Слой бизнес-логики (Business Logic Layer)

**Сервисы:**

**CalculationService** - выполнение медицинских расчетов
- `calculateBMRMifflin()` - формула Mifflin-St Jeor
- `calculateBMRHarris()` - формула Harris-Benedict
- `calculateTDEE()` - расчет суточной потребности
- `generateInterpretation()` - создание рекомендаций

**ValidationService** - валидация входных данных
- `validateWeight()` - проверка веса
- `validateHeight()` - проверка роста
- `validateAge()` - проверка возраста
- `validateAllInputs()` - комплексная валидация

**DatabaseService** - работа с базой данных
- `initDatabase()` - инициализация БД
- `saveCalculation()` - сохранение расчета
- `getCalculations()` - получение истории
- `deleteCalculation()` - удаление расчета

### 5.4 Слой данных (Data Access Layer)

**База данных SQLite:**

Таблица `calculations`:
- `id` - уникальный идентификатор
- `weight`, `height`, `age` - параметры пользователя
- `gender` - пол
- `activity_level` - уровень активности
- `formula_type` - используемая формула
- `bmr`, `tdee` - результаты расчетов
- `created_at` - дата и время расчета

### 5.5 Паттерны проектирования

1. **Singleton** - для DatabaseService (единственный экземпляр)
2. **Strategy** - для выбора формулы расчета BMR
3. **Repository** - для абстракции работы с данными
4. **DTO (Data Transfer Object)** - для преобразования данных между слоями

### 7.2 Экран расчета (CalculationScreen)

**Назначение:** Ввод параметров пользователя для выполнения расчета.

**Поля ввода:**
- Вес (кг) - числовое поле, диапазон 30-300
- Рост (см) - числовое поле, диапазон 100-250
- Возраст (лет) - числовое поле, диапазон 18-100

**Выбор параметров:**
- Пол: Мужской / Женский
- Уровень активности: 5 вариантов с коэффициентами
- Формула расчета: Mifflin-St Jeor / Harris-Benedict

**Действия:**
- Ввод данных
- Валидация при попытке расчета
- Выполнение расчета
- Переход к результатам

### 7.3 Экран результатов (ResultScreen)

**Назначение:** Отображение результатов расчета с интерпретацией.

**Отображаемая информация:**
- BMR (базовый метаболизм) - выделено цветом
- TDEE (суточная потребность) - выделено цветом
- Целевые калории:
  - Для поддержания веса
  - Для снижения веса
  - Для набора веса
- Параметры расчета (вес, рост, возраст, пол, активность)
- Рекомендации (список)
- Предупреждения (при наличии)

**Действия:**
- Просмотр результатов
- Переход к новому расчету
- Возврат на главную

### 7.4 Экран истории (HistoryScreen)

**Назначение:** Просмотр ранее выполненных расчетов.

**Отображаемая информация:**
- Список всех расчетов (от новых к старым)
- Для каждого расчета:
  - Дата расчета
  - BMR и TDEE
  - Параметры (вес, рост, возраст)

**Действия:**
- Просмотр списка
- Обновление списка (pull-to-refresh)

## 8. ИНСТРУКЦИЯ ПО ЭКСПЛУАТАЦИИ

### 8.1 Установка приложения

1. Скачайте APK файл приложения
2. На Android устройстве разрешите установку из неизвестных источников:
   - Настройки → Безопасность → Неизвестные источники
3. Откройте APK файл
4. Нажмите "Установить"
5. Дождитесь завершения установки
6. Запустите приложение

### 8.2 Выполнение расчета

**Шаг 1:** Запустите приложение, на главном экране нажмите "Новый расчет"

**Шаг 2:** Введите ваши параметры:
- Вес в килограммах (например, 70.5)
- Рост в сантиметрах (например, 175)
- Возраст в годах (например, 30)

**Шаг 3:** Выберите ваш пол (Мужской/Женский)

**Шаг 4:** Выберите уровень физической активности:
- Минимальная (1.2) - сидячий образ жизни
- Легкая (1.375) - легкие упражнения 1-3 раза в неделю
- Умеренная (1.55) - упражнения 3-5 раз в неделю
- Высокая (1.725) - интенсивные тренировки 6-7 раз в неделю
- Экстремальная (1.9) - ежедневные тренировки + физическая работа

**Шаг 5:** Выберите формулу расчета:
- Mifflin-St Jeor - рекомендуется для большинства людей
- Harris-Benedict - классическая формула

**Шаг 6:** Нажмите кнопку "Рассчитать"

**Шаг 7:** Изучите результаты:
- Ваш базовый метаболизм (BMR)
- Суточная потребность в энергии (TDEE)
- Рекомендации по калорийности для разных целей

### 8.3 Просмотр истории

1. На главном экране нажмите "История расчетов"
2. Вы увидите список всех ваших расчетов
3. Нажмите на любой расчет, чтобы просмотреть его детали
4. Потяните список вниз, чтобы обновить

### 8.4 Интерпретация результатов

**BMR (Basal Metabolic Rate):**
- Количество калорий, которое ваш организм сжигает в состоянии покоя
- Базовый уровень для расчета суточной потребности

**TDEE (Total Daily Energy Expenditure):**
- Общая суточная потребность в калориях
- Учитывает вашу физическую активность

**Целевые калории:**
- **Поддержание веса:** потребляйте калорий = TDEE
- **Снижение веса:** потребляйте калорий = TDEE - 500
- **Набор веса:** потребляйте калорий = TDEE + 500

### 8.5 Рекомендации по использованию

✅ **DO (Делайте):**
- Вводите точные данные для более точных результатов
- Регулярно обновляйте параметры при изменении веса
- Следуйте рекомендациям постепенно
- Консультируйтесь с врачом при значительных изменениях диеты

❌ **DON'T (Не делайте):**
- Не опускайтесь ниже минимальной калорийности (1200 ж/1500 м)
- Не используйте результаты как единственный источник рекомендаций
- Не игнорируйте предупреждения приложения
- Не применяйте результаты к детям и подросткам
